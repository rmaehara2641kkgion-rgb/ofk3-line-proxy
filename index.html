原因 ファイル内に addrMaster を使う場所が2つあります： 先に使われる場所（「住所キャッシュ管理」セクション） javascriptvar addressCache = {}; if (Object.keys(addrMaster).length > 0) { // ← ここで addrMaster を使う ... } 後で定義される場所（ずっと下の「MAP管理 - 住所マスター機能」セクション） javascriptvar addrMaster = JSON.parse(localStorage.getItem('addrMaster') || '{}'); JavaScriptは var 宣言を巻き上げます（hoisting）が、値の代入は実際にその行を実行するまで行われません。スクリプトは上から順番に実行されるので、①の時点では addrMaster はまだ undefined の状態です。そこで Object.keys(undefined) が呼ばれてエラーになっています。 修正方法 var addrMaster = JSON.parse(...) の行を、使われる場所より前に移動してください。 Step 1：下の方にあるこの行を探して削除 javascript // ===== MAP管理 - 住所マスター機能 ===== var addrMaster = JSON.parse(localStorage.getItem('addrMaster') || '{}'); ↓ javascript // ===== MAP管理 - 住所マスター機能 ===== （コメント行だけ残して、変数宣言の行を削除） Step 2：ファイル冒頭のState宣言部分（let lineProfileCache = ... の近く）に追加 javascript // LINE profile cache let lineProfileCache = JSON.parse(localStorage.getItem('lineProfileCache') || '{}'); var PROFILE_CACHE_TTL = 24 * 60 * 60 * 1000; // 住所マスター（MAP管理機能で使用、addressCacheより先に必要） var addrMaster = JSON.parse(localStorage.getItem('addrMaster') || '{}'); これで addrMaster が使われるより前に必ず値がセットされるようになります。 修正後、ファイル全体をもう一度貼っていただければ、他に同様の「先に使って後で定義」の問題がないかも確認できます。
